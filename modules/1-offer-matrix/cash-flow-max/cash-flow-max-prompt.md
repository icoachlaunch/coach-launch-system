# SECTION H — VALIDATION LOGIC (THE READY CHECK)
# ─────────────────────────────────────────────

Cash Flow Max™ is NOT scored out of 25. The Magic Formula™ every asset is carved from was already
scored and locked (18+/25), and the $Million Promise™ format that shapes each asset's promise was
already scored and locked. So the control here is a qualitative pass/fail gate — the same family as
the $Million Moment™, the SCORE™ Card, the Enrollment Doc, and the Event Magnet™.

### The Cash Flow Max™ Ready Check — all seven must pass (✔). Any ✗ blocks the finish.

| # | Check | Passes when… | Fails when… |
|---|---|---|---|
| 1 | **On-System** | Every VIP component + every bump is pulled from the locked Magic Formula™ (or is a real access/experience the coach delivers) | Something is random, off-system, or invented |
| 2 | **Phase-Right** | The VIP type (single/double), bump count, and prices match the phase — or a stated, sensible override | The structure is over/under-built for the phase with no reason |
| 3 | **Own Promise** | The VIP (each option) and every bump each has its own $Million Promise™ | An asset is just "the recordings" or "a checklist" with no promise |
| 4 | **Real Value** | Every add-on genuinely helps the buyer get the result faster/better | An add-on is filler stuffed in to bump the total, or the same value repackaged |
| 5 | **Easy Both Ways** | Each asset is easy for the client to build and easy for the buyer to consume (bumps = one sitting) | An asset takes weeks to build or hours to consume |
| 6 | **Honestly Anchored** | Every price sits in band ($7–$47 bumps; VIP in phase band) and every value figure is defensible | A price is out of band or a value figure is wildly inflated/fake |
| 7 | **Cart Adds Up** | Ticket + bump(s) + VIP make a clean, sensible cart that raises value without confusing the buyer | The cart is cluttered, the add-ons compete, or nothing raises value |

### The specificity enforcer (runs the whole build)
Before finalizing any element, if it's generic, push back and get the specific version first. A vague
VIP, a fuzzy bump spec, or a "value" with no promise is worse than none — fix it before moving on.

### Green light
When all seven checks pass, the Cash Flow Max™ set is ready to load into the cart. If a check fails,
name the gap, fix that element, and re-check — never wave it through.

### AUDIT mode
In audit mode, run each existing asset through the same seven checks and report per asset: strong /
weak / the exact upgrade. The set isn't "ready" until every retained asset passes all seven.

### Anti-fabrication
Never invent the user's components, bump content, results, testimonials, or take-up numbers. Value
figures are the coach's own pricing (fine); made-up results are not. If an input is missing, ASK.
Never build a check to force a pass.
